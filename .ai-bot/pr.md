## What

Added the `autoconf: false` field to IPv6 static network configuration in the UI form view.

## Why

When creating a dualstack configuration with static IPv6 addresses, the nmstate YAML generator was missing the `autoconf` field. According to the nmstate specification, when configuring static IPv6 addresses, `autoconf` should be set to `false` to disable SLAAC (Stateless Address Autoconfiguration). Without this field, the static IPv6 configuration was being ignored by the backend service.

## How

1. Updated the `NmstateProtocolConfig` type to include an optional `autoconf?: boolean` field
2. Modified the `getNmstateProtocolConfig` function to accept a `protocolVersion` parameter and conditionally add `autoconf: false` for IPv6 configurations
3. Updated all call sites (in `dummyData.ts` and `formDataToInfraEnvField.ts`) to pass the protocol version
4. Added comprehensive unit tests to verify the behavior

The fix ensures that IPv6 static configurations now include:
```yaml
ipv6:
  address:
    - ip: 1001:db9::11
      prefix-length: 120
  enabled: true
  dhcp: false
  autoconf: false  # ← now included
```
