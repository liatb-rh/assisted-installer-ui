# MGMT-19002: Add autoconf field for IPv6 in nmstate UI form view

> **Reviewer summary:** Changes focus on the nmstate YAML generation logic in `libs/ui-lib/lib/ocm/components/clusterConfiguration/staticIp/data/`. The fix adds `autoconf: false` to IPv6 protocol configurations when generating static network configuration from the UI form view.

## What

Added the `autoconf: false` field to IPv6 protocol configurations in the nmstate YAML generated by the UI form view for static network configuration.

## Why

When creating a dualstack configuration with static IPv6 addresses, the UI form view was not including the `autoconf: false` field in the generated nmstate YAML. This field is required for IPv6 static configurations to prevent automatic address configuration (SLAAC) from interfering with the static IP assignment. Without this field, the static IPv6 configuration is ignored.

## How

1. Updated the `NmstateProtocolConfig` type to include an optional `autoconf` field
2. Modified `getNmstateProtocolConfig` function to accept a `protocolVersion` parameter
3. Set `autoconf: false` only when the protocol version is IPv6
4. Updated all call sites in `formDataToInfraEnvField.ts` and `dummyData.ts` to pass the protocol version
5. Added unit tests to verify the behavior

- `libs/ui-lib/lib/ocm/components/clusterConfiguration/staticIp/data/nmstateYaml.ts`: Updated function to include autoconf for IPv6
- `libs/ui-lib/lib/ocm/components/clusterConfiguration/staticIp/data/formDataToInfraEnvField.ts`: Updated call sites to pass protocolVersion
