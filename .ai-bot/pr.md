> **Reviewer summary:** Modified `libs/ui-lib/lib/ocm/components/clusters/ClustersTable.tsx` to add an empty state message when filter criteria match no clusters. The fix follows the established pattern from `AITable.tsx` using `getColSpanRow` to display a PatternFly `EmptyState` component.

## What

Add a "No matching clusters" message when filters are applied but no clusters match the criteria.

## Why

When users apply filters (search string or status filter) in the Assisted Clusters list and no clusters match, the UI displayed an empty table without any explanatory message, leaving users uncertain whether the filter failed, the system had an error, or truly no clusters matched.

## How

- Imported `getColSpanRow` and `EmptyState` from the common components library
- Added a `displayRows` memoized value that checks if `sortedRows` is empty while `rows` has content
- When this condition is true, display an `EmptyState` component with a clear message explaining no clusters match the current filter criteria
- Used `getColSpanRow` to properly span the empty state across all table columns, following the established pattern from `AITable.tsx`
- Added conditional rendering for the actions column to avoid rendering it for the empty state row
